# chaika

Claude Code skill для работы с медицинскими личными кабинетами:

1. **Чайка** (`chaika.com`) — выгрузка истории визитов и скачивание документов (протоколы, акты, чеки).
2. **АльфаСтрахование** (`med.alfastrah.ru`) — подача скачанных документов на возмещение по ДМС с учётом номеров заявлений.

> ⚠️ Все персональные данные и сессии хранятся **вне репозитория** (в `~/.`-файлах с правами `600`). В репозитории — только обезличенный код и документация.

## Установка

Скилл подключается симлинком в каталог скиллов Claude Code:

```bash
git clone git@github.com:Dekkee/chaika.git ~/git/chaika
ln -s ~/git/chaika ~/.claude/skills/chaika
```

После этого правки в `~/git/chaika` сразу подхватываются скиллом — единый источник правды это репозиторий.

## Структура

| Файл | Назначение |
|------|------------|
| `SKILL.md` | Инструкции скилла: actions `visits` / `docs` / `details` / `refund`, флоу API, алгоритмы |
| `submit_alfastrah.py` | Обезличенный скрипт подачи заявлений на возмещение (без перс. данных) |
| `alfastrah_profile.template.json` | Шаблон профиля (имена полей, без значений) |
| `.gitignore` | Защита от коммита секретов, профиля с данными, HAR и рантайм-файлов |

## Конфигурация (вне репозитория, `chmod 600`)

| Файл | Что |
|------|-----|
| `~/.chaika_cookies` | Cookies сессии Чайки (`remember`, `session`) |
| `~/.chaika_ehr_id` | EHR ID пациента |
| `~/.alfastrah_cookies` | Cookies сессии АльфаСтрахования (снимать свежими — токен живёт ~5 мин) |
| `~/.alfastrah_profile.json` | Персональные данные для заявлений (паспорт, банк, ИНН/СНИЛС, ФИО, полис) — заполнить по `alfastrah_profile.template.json` |

Куки берутся из DevTools → Network → `Copy as cURL` любого запроса соответствующего сайта. Если файлов нет — скилл сам запросит их у пользователя.

## Использование

Через Claude Code (после установки скилла):

```
/chaika visits              # история визитов таблицей
/chaika docs 25.06.2026     # скачать документы визитов за дату в ~/Downloads/chaika/
/chaika details <entry-id>  # детали конкретной записи
/chaika refund              # подать визиты на возмещение ДМС
```

Скрипт подачи (вызывается скиллом, можно и напрямую):

```bash
# 1) сформировать список визитов к подаче (без перс. данных)
#    ~/Downloads/chaika/_alfastrah/claims.json:
#    [{"folder":"<папка визита>","date":"ГГГГ-ММ-ДД","description":"...","sum":1500}]

# 2) проверка без отправки
python3 ~/.claude/skills/chaika/submit_alfastrah.py ~/Downloads/chaika/_alfastrah/claims.json --dry-run

# 3) подача (создаёт РЕАЛЬНЫЕ страховые заявления)
python3 ~/.claude/skills/chaika/submit_alfastrah.py ~/Downloads/chaika/_alfastrah/claims.json

# 4) обновить статусы в таблице до актуальных с портала
python3 ~/.claude/skills/chaika/submit_alfastrah.py --sync
```

После успешной подачи скрипт:
- пишет `Заявление №<номер>.txt` в папку визита;
- добавляет строку в таблицу `~/Downloads/chaika/Заявления.csv` (визит → №заявления → сумма → статус);
- защищает от дубля (пропускает визит, если в папке уже есть маркер заявления).

Статусы заявлений: **Отправлено → На рассмотрении → Оплачено** (рассмотрение до 15 календарных дней).

## Как это работает (АльфаСтрахование)

- Авторизация — **только куками** (`accessToken` JWT, ~5 мин); CSRF-заголовок не нужен (хватает куки `CsrfToken` + same-origin).
- Подача = 2 запроса: `POST /api/policy/dms/{policy}/claim` (создаёт заявление, возвращает номер) → `PUT .../claim/{id}` с документами в виде **сырого base64** в поле `files[]`.

## Безопасность

- В репозитории **нет** персональных данных — только обезличенный код. Проверяется перед каждым коммитом.
- `.gitignore` блокирует случайный коммит кук, профиля с данными, HAR-файлов и рантайм-выгрузок.
- Секреты и медицинские документы остаются на локальной машине.
